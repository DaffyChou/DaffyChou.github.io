物料採購單價走勢
同一物料在不同廠商的採購單價變化（資料來自採購單與詢價單歷史）
要看什麼
① 不同廠商的價差：同一個月同樣物料，最高與最低報價差多少。 ② 每個月的漲跌走勢：價格穩定、上升還是下降。 ③ 單一廠商連續調漲：判斷是不是議價能力在流失。
什麼情況要警覺
單月漲幅超過 5%（可能是物價或匯率上漲被廠商轉嫁）； 某廠商比平均貴 20% 以上（議價力不足）； 廠商間價差超過 30%（規格或品質可能不同，要確認）。
建議怎麼做
啟動重新詢價、下次「綜合詢價單」強制找至少 3 家比價、 建立替代廠商名單；油料類波動大的品項可考慮長約鎖價。
同類產品做法
SAP Ariba 的「價格智能分析」、Coupa 的「支出守門」、 Sertica 的「歷史價格追蹤」、SpecTec AMOS 的「報價比對」。
配件更換頻率
全船隊 各品項/設備累計更換次數（資料來自領用單＋派工單），找出最常更換的關鍵配件
更換次數 累計佔比
要看什麼
排在前面的少數品項通常吃掉大部分的更換量，這些是維修預算和安全庫存的主要來源； 比對保養計畫的週期是否吻合（換得太頻繁 = 設備老化或廠商品質有問題）。
什麼情況要警覺
同一個配件 一個月換 3 次以上（懷疑品質瑕疵）； 派工次數 本季比上季多 30%（設備可靠度下降）； 更換量突然驟降但又出現缺料 → 可能是船端有故障沒報修。
建議怎麼做
納入預防性維修排程、向廠商提出品質改善通報、 重要配件改用雙廠商備援；與保養計畫連動強制更換週期。
同類產品做法
SpecTec AMOS 的「設備可靠度追蹤」、DNV ShipManager 的「保養計畫＋採購整合」、 Sertica 的「重要設備備品管理」。
庫存健康度
全船隊各品項的健康狀態 各品項的健康狀態 — 橫軸：安全庫存量　縱軸：現有庫存量　氣泡大小代表金額
缺料（庫存少於安全量） 警戒（不到安全量的 1.5 倍） 健康（安全量 1.5～3 倍） 過量（超過安全量 3 倍）
要看什麼
缺料區：影響船舶營運與適航； 警戒區：該補貨的觸發點； 過量區：資金被佔用、未來可能報廢； 氣泡愈大代表金額愈高，要優先處理。
什麼情況要警覺
缺料品項超過 10%（停航風險）； 過量金額超過庫存總值的 15%（呆料貶值）； 關鍵安全救生類物料出現在缺料區 → 法規不合規。
建議怎麼做
缺料 → 立即發出緊急請購＋詢價； 過量 → 船間調撥或退回岸上倉庫； 依消耗速度和廠商備貨天數，動態調整補貨警戒線與安全庫存量。
同類產品做法
Microsoft Dynamics 365 的「庫存健康總覽」、Sertica 的「庫存警示四象限」、 NetSuite 的「ABC 重要度＋健康度矩陣」。
盤點差異率
盤點差異絕對值 / 帳面數量， 按船舶與倉庫拆解按 的各倉庫拆解；紅色菱形是 3% 內控警戒線
要看什麼
哪艘船的哪個倉庫差異率最高 的哪個倉庫差異率最高、是否集中在某些品類； 對照負責人或船員交接時點，判斷是流程問題還是人為因素。
什麼情況要警覺
差異率 超過 3%（內控警戒線）； 連續兩季上升（系統性問題）； 某個倉庫明顯比船隊平均高 → 是流程或人員稽核重點。
建議怎麼做
提高該倉庫的盤點頻率（從季→月，或月→週）； 出入庫單據改成雙人核對；重要安全物料改成每次出庫即時盤點。
同類產品做法
美國 SOX 法規的「庫存準確性控制」、BASS 的「主資料品質」、 ABS NS5 的「盤點差異稽核」。
盤點差異金額
差異 × 庫存單價；正值代表盤盈、負值代表盤虧、折線是淨差異
要看什麼
盤盈與盤虧的金額對比，以及淨差異的走勢； 這些金額會直接影響損益表。
什麼情況要警覺
當月淨盤虧超過 10 萬新台幣（重大異常）； 連續三個月都盤虧（可能系統性流失或失竊）； 盤盈金額異常大 → 過去可能有入庫漏記或品名錯置。
建議怎麼做
啟動責任歸屬調查、追溯出入庫單據； 金額重大時開立調整單並通報內稽；連續異常的列入年度稽核重點。
同類產品做法
SAP S/4HANA 的「庫存成本差異」、Oracle Cloud 的「盤點調整」、 Sage X3 的「短缺報表」。
供應商集中度
廠商各自的採購／詢價金額佔比；用集中度指數判斷是否過度依賴單一廠商
集中度指數 = 0
要看什麼
第 1 名與前 3 大廠商各自的金額佔比、整體集中度指數 （低於 1500 屬市場競爭、1500～2500 中度集中、超過 2500 高度集中）。
什麼情況要警覺
單一廠商占比 超過 30%（依賴風險，廠商一斷供就停擺）； 集中度指數 超過 2500（談判時居劣勢）； 關鍵安全或主機物料沒有備援廠商（被綁住）。
建議怎麼做
拓展次要供應商名單、設定單一廠商占比上限（例如不超過 25%）、 重要物料強制至少找 3 家報價；長期建立雙廠商或三廠商備援結構。
同類產品做法
SAP Ariba 的「供應商風險與集中度」、Coupa 的「風險評估」、 Wilhelmsen 的「供應商整合管理」。
庫存帳齡分布
全船隊 依最近一次入庫或採購日期推算庫齡；金額分為 0-30 / 31-90 / 91-180 / 181-365 / 365 天以上 五段
要看什麼
各帳齡區段的金額佔比，超過 180 天的就是呆滯料候選； 對照消耗速度，評估是否要提列存貨跌價損失。
什麼情況要警覺
365 天以上的庫齡佔比超過 10%（明顯呆料）； 181～365 天區段連續成長（即將惡化）； 整體庫存周轉天數超過 180 天（資金使用效率太差）。
建議怎麼做
提列存貨跌價損失、啟動處分／退廠／促銷程序、 修正下次請購數量；快過期的物料優先船間調撥消化。
同類產品做法
NetSuite 的「庫存帳齡」、Sage X3 的「滯銷分析」、 Sertica 的「呆滯料報表」、Oracle Cloud 的「帳齡區間分析」。
交船庫存差異
全船隊 相關的船舶交接或跨船調撥時的物料金額流動與差異（資料來自調撥單與盤點計畫）
要看什麼
跨船東或船舶的調撥金額流向、交接當時的實盤與帳面差異； 判斷是否需要跨公司計費或開立請款單。
什麼情況要警覺
跨船東調撥沒有開立請款單（跨主體計費風險、稅務風險）； 交接差異 超過 5%（責任歸屬可能有爭議）； 離船員工最後盤點沒完成就交接。
建議怎麼做
標準化交船清單與雙方簽認流程；跨船東調撥一律開立請款單； 交接前 7 天內必須完成完整盤點並追蹤差異。
同類產品做法
Veson IMOS 的「船舶交接」、Wilhelmsen 的「交接流程規範」、 DNV ShipManager 的「船舶異動管理」。
