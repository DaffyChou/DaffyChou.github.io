## 1. 系統目的

持續比對「船舶 AIS 實際動態」與「航行計畫(StormGeo / WNI)」,於航線偏移、速度異常、錨泊漂移、外部警示信件等風險發生時主動告警,並以事件工作流追蹤處理至結案。功能含五頁:船隊監控、單船詳細、事件處理、警示時間線、閾值設定。

## 2. 系統整合

| 介面 | 方式 | 重點 |
|---|---|---|
| 戰情室·監管人員管理系統 | 同步 | 人船對應之唯一來源;改派後可見範圍、警示計數、事件指派同步反映;同步異常時顯示狀態並以最後成功結果運作 |
| 戰情室·權限管理 | 同步 | 監管人員以外之使用者(主管、岸勤等)的檢視/編輯授權;粒度=功能模組 × 動作 × 資料範圍 |
| StormGeo 航行計畫 | **API** | 排程/觸發自動拉取匯入 |
| WNI 航行計畫 | **信件** | 解析信件與附件(計畫書/航路點表/ECDIS 航線檔);失敗通知+人工補登;日後可平移 API |
| AIS 資料服務 | 介接 | 位置、SOG/COG、航行狀態、歷史軌跡、逐點偏移 |
| 警示信件 | 信件 | 解析事件時間、代碼、等級(Warning/Information)、描述、位置 |

計畫匯入後統一結構:計畫編號、來源、匯入時間、航路點(座標/類型/預定時間/港口)、**各航段配速**、Safety Frame。

## 3. 角色與權限

| 角色 | 來源 | 範圍 |
|---|---|---|
| 監管人員 | 戰情室監管人員管理系統 | 僅自己被指派之船舶(全畫面一致限縮) |
| 主管 | 戰情室權限管理 | 全船隊,可識別各船監管 |
| 其他使用者 | 戰情室權限管理 | 依授權範圍;檢視/編輯分離,僅檢視者隱藏所有編輯控制項 |

編輯權對應:事件(狀態轉換含批次、改派、備註)、閾值(預設值、批次/單船覆寫、漂航圈半徑)。

## 4. 風險監測(4 維度)

| 維度 | 判定 | N/A |
|---|---|---|
| 航線偏移 | max(目前 XTD, 歷史最大偏移) ≥ 閾值(NM);預設 W 1.5 / C 3.0 | 港內暫停 |
| 速度異常 | (SOG−配速)÷配速 > 閾值;**僅超速觸發**;預設 W 20% / C 40% | 無 SOG/配速 |
| 漂航圈 | 錨泊時船位至圈心距離 > 半徑;預設 W 500 / C 750 m;圈心=計畫到港點 | 非錨泊 |
| 警示信件 | 有 Warning 信→Critical;僅 Information→Warning | 無信件 |

整體風險=最嚴重維度(N/A 不計)。閾值層級:**單船覆寫(可分航段)> 全船隊預設**,依當前航段套用;變更即時重算並補立事件。

**配速邏輯**:一律取計畫之航段配速,依序:① 船位投影所在航段 → ② 時程區間所在航段 → ③ 全航程平均(>0 之航段)→ ④ 皆無則不判定。配速 0 之節點不判定;VSR 低速段以該段配速為準。

**錨泊提醒**:AIS 轉為錨泊(拋錨/漂航)即發提醒警報,要求監管設定漂航圈;設定/確認前以預設半徑暫行監測;完成後提醒解除,離錨自動標示解除。

## 5. 事件管理

- 自動立案:任一維度達 Warning/Critical;粒度=一船×一維度×當月一案,警示信件**一封信一案**;錨泊設定提醒亦立案。
- 狀態機(三狀態):**新立案 New → 處理中 In Progress → 已解決 Resolved**;可退回/重開;舊「已認可」資料併回新立案。
- 條件消失:標示「警報已解除」,**不自動結案**;已解決後再觸發自動重開。
- **批次複選**:逐筆勾選/全選,一次轉換多筆狀態;逐筆寫入歷程;僅編輯權者可見。
- 每筆事件:歷程(狀態/時間/操作者)、備註累積、自動指派該船現任監管、可改派。
- 清單:統計 New/In Progress/Resolved;篩選狀態/等級/維度/船舶;排序開立時間或狀態(新立案>處理中>已解決);一鍵地圖定位(漂航圈→圈心、偏移→最大偏移點、信件→信件位置、其餘→船位)。

## 6. 頁面功能

**船隊監控(預設)**:上地圖下列表(30%↔50% 可調、與篩選同步)。船標依 COG 旋轉、風險著色帶脈衝;篩選=風險/AIS 狀態/關鍵字;警示面板列出需關注船舶可點跳;圖例含即時閾值。列表欄位:船舶、AIS 狀態(避碰三色)、整體風險、4 維度矩陣、計畫來源徽章、進度、偏移量(現值/最大/Safety)、回報時間(UTC+8)、SOG/HDG/經緯度(DMS);依風險排序。點船進入單船檢視:地圖(計畫航線、Safety Frame 走廊、AIS 軌跡正常/偏移分色、船位、信件標記,各層可開關)+ 面板(XTD 讀值與 70% 警戒、警示與事件列、AIS 現況、計畫資訊)。

**單船詳細頁**:獨立網址,同上加圖例與 XTD 三標尺;信件標記點開顯示全文。

**警示時間線**:信件+系統事件合併倒序,依可見範圍過濾。

**閾值設定**:兩模式——「分航段集體設定」(選航段+套用對象,多維度即輸即存)與「單船設定」(選船+航段,顯示**船舶現況**即時預覽徽章,單維度可還原)。航段=海上(航線偏移★/速度)、錨泊(漂航圈★)、全航段。驗證 C>W;漂航圈半徑亦可由警示列/地圖圈心直接設定。

## 7. 地圖與 RWD

- 跨換日線完整連續顯示(航線/走廊/標記),換日線兩側船舶同視野可見。
- 漂航圈雙環(內 W 橘/外 C 紅),越界強化並繪船位→圈心連線,圈心點擊調半徑。
- 底圖暗色/海圖風格切換;座標 DMS;時間 UTC 儲存、UTC+8 顯示。
- ≤1024px:列表/事件/閾值卡片化,三段檢視(列表/分割/地圖),篩選可收合。
- 全站中英雙語、四色風險一致、鍵盤可操作、介面偏好保留。

## 8. 本期未實作(預留)

海氣象/AIS 訊號中斷/主機異常維度、WNI API 介接、閾值依航段自動切換生效、計畫帶入之預設閾值。

## 9. 驗收要點

1. 戰情室改派後可見範圍/計數/指派同步;監管不可見非指派船。
2. 僅檢視權者無任何編輯控制項(含複選框)。
3. StormGeo API、WNI 信件匯入後欄位與監控邏輯一致;解析失敗有通知。
4. 偏移達 1.5/3.0 NM 同步告警立案;VSR 段配速 10 節時 SOG 9 節不告警、13 節告警 30%。
5. 轉錨泊即出現設定漂航圈提醒;設定前以預設半徑暫行監測;偏離 500/750 m 觸發。
6. 事件三狀態流程完整、可批次轉換並逐筆留歷程;條件消失標「警報已解除」不自動結案。
7. 跨換日線航線各縮放層級連續;行動版卡片化正常。
