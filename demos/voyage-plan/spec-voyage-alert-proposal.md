## 1. 系統概述

### 1.1 系統目的與整合定位

**整合定位**：本案功能將整合進貴公司既有的**戰情室系統**內，作為其中一個模組存在，並沿用戰情室原本的地圖底圖圖層、視覺風格與既有的使用者登入機制（不會另建獨立系統）。

**系統目的**：針對船公司營運中心（OCC）日常監控船隊航行安全的場景，提供單一介面整合：

- 整支船隊位置、風險等級、是否進入高風險區域的即時視覺化
- 從多個獨立維度量化每艘船的安全狀態
- 自動把每筆 Critical / Warning 立案，由值班人員以標準流程處理
- 可調整觸發閾值，依風險偏好客製化警示靈敏度

### 1.2 預期效益

| 效益 | 說明 |
|---|---|
| **單一畫面掌握船隊狀態** | 以往需要切換 AIS 平台、StormGeo Web、Email 信箱、風險地圖等多套系統，整合到一個視窗 |
| **降低漏報風險** | 自動立案 + 狀態流轉確保每筆警示都有人接手、不會被信件淹沒 |
| **加速應變決策** | 單船 detail 直接看到 XTD 偏移、警示信件全文、進行中事件，免再查多處 |
| **沉澱處理知識** | 每筆事件有完整備註與狀態歷程，可作後續檢討、稽核、知識傳承 |
| **可配置的觸發條件** | 不同公司、不同航線、不同船型可調整 Warning / Critical 閾值 |

### 1.3 使用對象

| 角色 | 主要使用情境 |
|---|---|
| 值班操作員 (Operator) | 開啟船隊總覽，掃描需關注船舶 → 點進單船 detail 確認狀況 → 切到事件處理頁做立案流轉 |
| 船長 / 海務 (Captain / Maritime) | 接案、新增處理備註、查看時間線回顧 |
| 系統管理員 / Supervisor | 依公司風險偏好調整閾值，存檔後立即重評估全船隊 |

## 2. 功能模組總覽

系統包含 6 個主要模組，分別位於左側導覽列：

### 2.1 船隊監控

- 上方：世界地圖，每艘船以箭頭 icon 顯示（顏色＝風險等級、方向＝航向 COG）
- 下方：船舶列表，含多維度警示熱圖、計畫進度、偏移量、AIS 即時資料
- 點任一船舶切換為單船 detail 模式（上方變為單船地圖 + 右側資訊面板）
- 篩選：全部 / 需關注 / Critical / Warning / Normal；可依船名或 IMO 搜尋
- Critical Banner：當有 Critical 船舶時右上跳出醒目提示，可手動隱藏

### 2.2 單船 Detail

- 單船地圖含 5 個可切換圖層：計畫航線（含 Waypoints）、Safety Frame 走廊、AIS 軌跡（偏移段紅色）、船舶目前位置、警示信件標記
- 右側資訊面板：XTD 即時讀值（含進度條）、各維度警示明細、進行中事件、AIS 位置、計畫航線、Open Event 詳情
- 跨換日線航程正確繪製，全球視角不會脫節

### 2.3 事件處理

- 系統依照當前船隊警示自動立案，每艘船 + 每個維度 + 每月最多一案，避免重覆
- 標準狀態流程：新立案 (New) → 已認可 (Ack) → 處理中 (In Progress) → 已解決 (Resolved)，已解決可重新開啟
- 可依狀態、等級、維度、船舶四個條件交叉篩選
- 點任一事件開 Modal：可指派操作員、新增備註、查看狀態歷程
- 觸發條件已消失但事件未結案時自動標記 cleared，提示操作員後續處理

### 2.4 警示時間線

- 把各來源的警示信件（StormGeo 通知信、其他維度觸發信件）與進行中事件混合，依時間倒序呈現
- 顏色區分等級：Critical 紅、Warning 橘、Information 藍
- 用於回顧、晨會檢討、事故分析

### 2.5 閾值設定

- 提供可調整維度的 Warning / Critical 觸發值
- 修改後立即重評估全船隊、自動補立或標記消失的事件
- 與預設值不同時顯示「已修改」標記，可一鍵 Reset

### 2.6 全球風險區圖層

- 8 大類別 polygon 顯示於地圖：海盜、戰爭、制裁、軍演、氣象、ECA、保護區、壅塞
- 每類別可獨立開關，並提供「全開 / 全關」捷徑
- 點任一區域顯示 popup：類別、名稱、風險等級、生效期間、邊界座標、資料來源連結

## 3. 警示維度與分期規劃

### 3.1 維度總覽

系統設計 9 個警示維度，每艘船在每個維度都會被獨立評估為 Critical / Warning / Normal / N/A 四種狀態，整體風險取最嚴重等級。

| 維度 | 中文名 | 監測目的 |
|---|---|---|
| route | 航線偏移 | 是否偏離計畫航線（XTD 對比 Safety Frame） |
| alert | 警示信件 | 彙整來自各監測來源（StormGeo、其他計畫服務商、route / speed / comm / zone / sea / engine 等維度觸發）的通知信件 |
| speed | 速度異常 | 對地航速是否與計畫平均嚴重不符 |
| comm | AIS 訊號 | AIS 訊號是否中斷或延遲 |
| port | 到港/離港 | 進出港敏感期是否異常 |
| event | 進行中事件 | 彙整各維度（route / speed / comm / zone / sea / engine 等）尚未結案的監測事件，含 StormGeo 上游推送與系統自身偵測 |
| zone | 區域警示 | 船舶當前位置是否落入高風險區 |
| sea | 海況/天候 | 浪高、風力是否超標 |
| engine | 主機異常 | RPM、排氣溫、油壓是否在規格內 |

### 3.2 Phase 1 範圍（本案）

| 維度 | 範圍 | 資料來源 | 備註 |
|---|---|---|---|
| **route** | ⭐ 主推 | 計畫航線、AIS | 本次提案核心功能 |
| **alert** | ⭐ 主推 | StormGeo 通知信件、其他各監測維度觸發信件 | 本次提案核心功能 |
| speed | ✅ 同步交付 | AIS | — |
| comm | ✅ 同步交付 | AIS | — |
| port | ✅ 同步交付 | 計畫航線、AIS | — |
| event | ✅ 同步交付 | StormGeo Open Event、各監測維度產生之事件 | 含上游推送與系統自身偵測 |
| zone | ⚙ 需協助 | 由客戶提供查看資料的網址，由 SLAB 進行爬文整理 | 上線後可再補充類別 |

> 所有維度的 Warning / Critical 觸發點皆由「閾值設定」頁設定，本表只列資料來自哪裡。

> **計畫航線資料來源說明**：系統設計支援多家計畫服務商，目前包含 **StormGeo** 與 **WNI** 兩家。本案 Phase 1 已收到貴公司提供的 StormGeo API，WNI 對接窗口尚未取得，**Phase 1 先以 StormGeo 為主**，WNI 接入待後續取得 API 資料後再行排程（系統架構已預留多計畫源支援）。

### 3.3 Phase 2 / 加購選項

| 維度 | 條件 | 備註 |
|---|---|---|
| sea | 需確認氣象資料供應商 | 候選：NOAA / WMO / JMA / 商業氣象 API；費用、合約由客戶或我方任一方負責，需另行討論 |
| engine | 需船上 IoT 感應器串接 | 由客戶 IT 配合提供 RPM、排氣溫、油壓等遙測通道；可隨船隊 IoT 計畫展開分批上線 |

> Phase 1 範圍與 Phase 2 條件成立後的擴充項目，建議分次報價並列於合約附件，避免後續變更。

### 3.4 整體風險判定規則

- 任一維度為 Critical → 整體風險 Critical
- 否則任一維度為 Warning → 整體風險 Warning
- 其餘 → Normal
- 例外：alert 維度若該船完全沒有任何警示信件紀錄，會顯示 N/A，不影響整體風險

## 5. 系統資料模型

以下為系統運作所需的核心資料結構，作為後續資料對接欄位 mapping 的參考。

### 5.1 船舶基本資料（Vessel）

| 欄位 | 說明 | 來源 |
|---|---|---|
| 船舶 ID | 系統內部識別碼 | SLAB 系統 |
| 船名 / IMO / MMSI | 國際識別碼 | 客戶提供 |
| 緯度 / 經度 | 最新位置 | AIS |
| 對地航速 SOG | 節（kn） | AIS |
| 對地航向 COG | 0–359° | AIS |
| AIS 時間戳 | UTC | AIS |
| AIS Nav Status | ITU-R M.1371 navigation status code | AIS（可選） |
| 當前 XTD | 偏離計畫航線的垂直距離（海浬） | 系統計算 |
| Safety Frame 寬度 | 由「閾值設定」頁設定，可全船隊統一或依船別/航線個別調整 | 系統內設定 |
| 航行進度 | 0–100% | 系統計算 |
| 最近計算時間 | 系統內 | SLAB 系統 |
| 計畫 ID / 來源 / 匯入時間 | 對應 StormGeo 計畫 | StormGeo |

### 5.2 計畫航線（Planned Track）

來源：**StormGeo（Phase 1 已對接）+ WNI（API 待提供）**，系統設計為多服務商可同時接入。

每筆航點：
- 序號 / 緯度 / 經度
- 該段 Safety Frame 寬度
- 計畫時間（UTC）
- 點位類型：Departure / Waypoint / Arrival
- UN/LOCODE 港口代碼、港名（若為港口點）
- 計畫航速 / 計畫航向

每艘船的「計畫來源」欄位會註明該筆計畫是來自 StormGeo 或 WNI，方便操作員辨識資料來源。

### 5.3 AIS 軌跡（AIS Track）

每筆位置點：
- 序號 / 緯度 / 經度
- SOG / COG
- AIS 時間戳
- 偏移距離（NM）
- 是否視為偏移點

### 5.4 進行中事件（Open Event）

事件來源涵蓋：
- **StormGeo 等計畫服務商上游推送的事件**（例如航線偏移 open_event）
- **系統自身依各維度監測自動立案的事件**（route / speed / comm / zone / sea / engine 等任一維度達 Warning / Critical 都會立案）

每筆事件包含：
- 事件 ID
- 觸發維度（route / speed / alert / ...）
- 等級：Critical / Warning
- 開始時間 / 最後偵測時間
- 與該維度相關的量化資料（例如 route 維度會記錄最大偏移距離 NM）
- 處理狀態與歷程（詳見 5.7）

### 5.5 警示信件（含 StormGeo 通知信與其他維度觸發信件）

| 欄位 | 說明 |
|---|---|
| 信件主旨 / 收信時間 | — |
| 船名 / IMO | — |
| 事件時間 / 警示代碼 / 等級 / 描述 | StormGeo 結構化欄位 |
| 信件原文節錄 | 可選擇是否在 Modal 中顯示完整內容 |
| 提及的港口代碼清單 | 自動匹配計畫航線 |
| 是否為離港 / 到港事件 | — |
| 經緯度（若可匹配） | 用於地圖標記 |
| 匹配的計畫航線點類型 / 港口代碼 / 港名 / 時間 | 自動或人工關聯 |
| 匹配方式 | port-departure / time-match / no-planned 等 |

> StormGeo 信件原文由 SLAB 開發解析模組（已有處理規則，正式版會持續優化）。

### 5.6 風險區（Risk Zone）

| 欄位 | 說明 |
|---|---|
| 類別 | 8 大類別之一 |
| 區域代碼 / 名稱（中英） | — |
| 風險分級 | critical / high / medium / low / info |
| 次類型 / 中英描述 | — |
| 資料來源 / 來源連結 | 公開資料來源 |

| 欄位 | 說明 |
|---|---|
| 事件 ID | `船 ID-維度-年月` 格式（避免同船同維度同月重覆立案） |

| 狀態 | New / Ack / In Progress / Resolved |

### 5.8 閾值設定

| 維度 | 單位 | 預設 Warning | 預設 Critical |
|---|---|---|---|
| 航線偏移 | × Safety Frame | 0.70 | 1.00 |
| 速度異常 | × 計畫平均速度 | 0.20 | 0.40 |
| AIS 訊號 | 通訊延遲分數（0–100） | 75 | 92 |
| 到港/離港 | 進出港敏感分數（0–100） | 65 | 88 |
| 海況/天候（Phase 2） | 海況指數（0–1） | 0.65 | 0.85 |
| 主機異常（Phase 2） | 主機分數（0–100） | 78 | 93 |

**Safety Frame 寬度設定**

Safety Frame 寬度（單位：海浬 NM）也在閾值設定頁中設定，可全船隊統一給定預設值，亦可針對個別船舶或航段覆寫。

> 預設值為依產業常用基準訂出的初步建議，Phase 1 上線前會與貴公司海務確認最終預設值。

## 7. 資料免責與法規說明

### 7.1 風險區邊界精度說明

系統內顯示的風險區邊界為**近似多邊形**，僅供介面視覺輔助與航前簡報參考，**不可作為法律或合規認定依據**。實際決策應以各權威來源公告為準（IMB / JWC / OFAC / 各國海事局 NAVAREA / IMO MEPC 等）。

### 7.2 第三方資料源即時性說明

- StormGeo 計畫航線、Warning Mail 之即時性與正確性，依其服務條款為準
- AIS 資料可能因海上訊號中斷、天氣、AIS 偽訊（spoofing）等因素出現延遲或失真
- 風險區資料來自公開資料源，更新頻率依各來源公告時間（系統至少每日同步一次）
- 系統呈現之警示為「依當前可得資料計算之提醒」，**不取代船員、船長、海務的專業判斷**

### 7.3 法規合規責任界面

- 系統提供風險區視覺化、警示提醒、事件流程紀錄等工具
- 是否避開風險區、是否遵守特定航線管制、是否申報相關文件等**最終合規責任，仍在貴公司及船舶營運方**
- 系統可作為內部稽核與決策紀錄依據，但**非取代海事法規與公司 SOP**
