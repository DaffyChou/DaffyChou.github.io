# 警報設定介面 — 測試計畫

**對象系統**：摯陞 SRP 船隊戰情室監控系統 — 警報設定 Demo (`alert_demo.html`)
**版本**：客戶版 (`IS_INTERNAL = false`) / 內部版 (`IS_INTERNAL = true`)
**測試範圍**：純前端單頁應用（HTML + jQuery + Select2），無後端 API；資料皆為示意（`SHIPS` / `USERS` / `CONFIG`）。

---

## 一、介面功能盤點（測試基準）

| 區塊 | 主要邏輯 |
|---|---|
| 三層連動 | 警報類型 → 警報欄位 → 觸發條件，逐層動態重建 |
| 警報類型 | 航次（7 欄位，含 2 個內部專屬隱藏欄位 `spd_rpm`、`zone`）、CII（2）、Bunker（2）、船員（3） |
| 觸發條件 | 依欄位 `parts` 動態產生 label／select／number 元件 |
| 警報範圍 | 航次/CII/Bunker → 全船隊 / 單船（Select2 複選）；船員 → 全體 / 個人 / 職等 |
| 偵測頻率 | 定時 / 每天 / 每周 / 每月（含星期挑選器） |
| 警報頻率 | 僅一次 / 重複提醒 / 每日固定 / 週報 / 月報 |
| 頻率預設 | 切換欄位自動套用 `FREQ_PRESETS` 偵測＋警報組合 |
| 通知設定 | 系統內部使用者（Select2 複選）＋ 自訂 Mail（tag input） |
| 執行摘要 | 右側即時反映所有設定 |
| 模式切換 | 內部版額外顯示觸發說明、設計理由、詳細說明卡、AI 研判、隱藏欄位 |
| Email 預覽 | 依當前設定動態生成（table-based + inline CSS），含解除通報連結 |

## 二、Unit Test（函式層級）

針對純函式與單一 DOM 行為，建議以 Jest + jsdom 或 Vitest 隔離測試。

### 2.1 描述字串生成函式

| 函式 | 測試重點 |
|---|---|
| `getDetectDescription()` | 四種 mode（scheduled/daily/weekly/monthly）各自輸出格式正確；weekly 多選與「（未選擇）」分支；空值套用預設（`00:00`、`6`、`1`） |
| `getFreqDescription()` | 五種 mode 輸出正確；`repeat` 帶間隔/單位/上限；星期與月日帶入正確 |
| `getScopeDescription()` | 全船隊回 `全船隊`；單船無選擇回 `（未選擇船舶）`；複選以「、」串接；船員三模式（全體/個人/職等）正確 |
| `getNotifDescription()` | 僅內部、僅 Mail、兩者、皆未勾選 → `（未設定）`；Mail 勾選但未填 → `（未填寫）` |
| `genericDesc(field)` | select 產生 `[A／B]`、number 產生 `[數值]` 佔位 |

### 2.2 狀態與預設

| 函式 | 測試重點 |
|---|---|
| `buildFieldList()` | 客戶版過濾 `hidden` 欄位（航次只剩 5）、內部版顯示全部（7）；`state.fieldKey` 設為第一個可見欄位 |
| `buildTrigger(field)` | 依 `parts` 產生正確元件數與型別；`state.vals` 以各部件預設值初始化；select 預設取 `opts[0]` |
| `applyFreqPreset(fieldKey)` | 每個欄位套用 `FREQ_PRESETS` 後，偵測/警報 mode 與子欄位值與設定表一致；無對應 key 時不報錯 |

### 2.3 Email 標籤輸入

| 函式 | 測試重點 |
|---|---|
| `addEmail()` | 去除前後空白與尾逗號；重複信箱不重複加入；空字串不加入 |
| `removeEmail()` | 正確移除指定項；不存在時不影響陣列 |
| `escapeHtml()` | `& < > "` 正確轉義（XSS 防護，因標籤以 `innerHTML` 插入） |

### 2.4 Email 模板

| 函式 | 測試重點 |
|---|---|
| `emailTpl()` / `EMAIL_TEMPLATES[key]` | 每個欄位皆能生成 `subject` 與 `body`，無未定義錯誤；`dataRows` 為空時略過資料區 |
| `unsubBlock(fieldKey)` | 僅「持續發送類」（alert.mode ≠ `once`）顯示解除連結；`gate`、`sea` 等一次性欄位不顯示 |
| `aiBlock()` / `actionsBlock()` | `IS_INTERNAL = false` 時一律回空字串；內部版且有資料時才渲染 |
| `getSampleShipName()` / `getRecipientsList()` | 單船有選 → 取第一艘；船員型 → `—`；無收件人 → `（尚未設定收件人）` |

## 三、Integration Test（模組互動／連動）

驗證跨元件的事件連鎖，建議以 Testing Library + jsdom 模擬使用者事件後檢查 DOM。

1. **類型切換連鎖**：點 tab 或改下拉 → `switchType()` 應依序重建欄位清單、觸發條件、套用頻率預設、切換範圍 UI、更新摘要與欄位列表標題（「共 N 個可用欄位」）。
2. **欄位切換連鎖**：改「警報欄位」→ 重建觸發條件 + 套用該欄位預設頻率 + 摘要同步；警報名稱 placeholder 自動更新為「欄位名稱：觸發描述」。
3. **觸發條件 ↔ 摘要**：改 select / number 輸入 → `state.vals` 更新 → 摘要「何時觸發」即時變動。
4. **範圍連動**：全船隊/單船切換 → 船舶複選框顯示/隱藏 + 摘要更新；船員型切到「個人/職等」→ 對應標籤區顯示且摘要文字改變。
5. **頻率子面板**：偵測/警報 mode 切換 → 僅對應 `.mode-sub` 顯示，其餘隱藏；子欄位值改動即時反映摘要；星期挑選器多選/取消同步「偵測頻率」描述。
6. **通知連動**：勾選「系統內部使用者」/「自訂 Mail」→ 對應區塊展開 + 摘要「通知對象」更新；Select2 與 tag input 變動皆觸發 `render()`。
7. **Email 預覽串接**：設定齊全後按「預覽 Email」→ modal 收件人 = `getRecipientsList()`、主旨/內文對應當前欄位模板、觸發時間填入；單船選擇會帶入船名。
8. **內部/客戶版一致性**：切換 `IS_INTERNAL` 重新載入後，觸發說明、設計理由、詳細說明卡、隱藏欄位、AI 研判/建議動作的顯示與隱藏需全域一致，無殘留。
9. **初始化**：頁面載入 `switchType('voyage')` → 預設選「進／出港狀態」、系統管理員預設勾選、定時頻率面板顯示。

## 四、UX / 操作測試要求

聚焦真實使用者操作、相容性與防呆，需人工＋跨環境驗證。

### 4.1 操作流暢度與回饋
- 所有設定變動，右側「執行摘要」**即時更新**、無明顯延遲或閃爍。
- 警報名稱 placeholder 隨設定動態提示，使用者可覆寫。
- 欄位列表中目前選取列以 `active-row` 高亮，與選擇一致。

### 4.2 鍵盤與互動
- 自訂 Mail：`Enter` 與逗號皆可確認；`Backspace`（輸入框為空時）刪除最後一筆；失焦自動加入未送出值。
- Email 預覽 modal：點 ×、點遮罩、按 `Esc` 三種方式皆可關閉。
- Select2：可搜尋、複選、`allowClear` 清除、無結果顯示「查無結果」、`closeOnSelect:false` 連續複選。

### 4.3 響應式與視覺
- 寬度 < 900px 時雙欄轉單欄（`grid-template-columns:1fr`）；船舶複選框 `max-width:100%` 不溢出。
- 暗色主題下文字／輸入框（白底）對比足夠、可讀。
- tabs 在窄螢幕可換行不破版。

### 4.4 邊界與防呆（建議補強項）
- 數值欄位輸入空值、0、負數、超大值（如偵測間隔、最大次數）時的顯示與後續行為。
- 自訂 Mail 輸入**非法格式**（缺 @、純文字）目前不驗證 → 建議加入格式檢查與提示。
- 單船模式未選任何船、勾 Mail 未填、未選任何收件人即按 Confirm/預覽 → 目前僅顯示「未選擇/未設定」文字，建議於送出時阻擋並提示。
- 每周偵測未選任何星期 → 摘要顯示「（未選擇）」，送出時應視為無效。
- 月日上限為 28（避免 29–31 在小月不存在）需於說明中明示。

### 4.5 Email 相容性
- 預覽內文採 table-based + inline CSS，需在 **Gmail、Outlook（桌面/Web）、Thunderbird、行動裝置** 實際寄送驗證版面不破。
- 解除通報為連結（非 JS 按鈕），確認各客戶端可點且 URL 正確帶 token。
- 變數標記（`var-tag`）、徽章、AI 提示框在客戶端的呈現。

### 4.6 跨瀏覽器
- Chrome / Edge / Firefox / Safari 主流版本；CDN（jQuery、Select2）載入失敗時的降級行為。

## 五、測試環境、缺陷分級與通過標準

**環境**：本機 Live Server（目前 `http://172.9.172.221:5501`）；需可連外載入 CDN 資源。分別以客戶版與內部版各跑一輪。

**缺陷分級**：
- P0 阻斷：連動中斷、JS 例外、Email 無法生成。
- P1 嚴重：摘要與設定不符、版本顯示錯誤、Email 版面破版。
- P2 一般：防呆缺失、邊界值處理不佳。
- P3 輕微：文案、對比、間距。

**通過標準**：Unit/Integration 案例全綠且無 console error；UX 無 P0/P1；P2 以上缺陷皆記錄並排程修補。
