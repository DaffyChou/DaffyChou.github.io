UI 整合設計選項 「週邊態勢」與「CPA」該不該分兩個介面？

問題：週邊態勢看「現在周圍有誰」，CPA 看「會不會碰」—— 兩者是同一份周圍監控資料， 只是 CPA 多了一層警示計算。答案：不該分兩個獨立介面，應為「一個介面 + 圖層切換」。 以下兩個選項都符合這個原則，差別在「整合的深度」。

選項 A：完全合一 —— 一個介面，圖層 toggle 切換

左側 Sidebar 只有「周圍態勢」一項；地圖上用圖層 toggle 顯示 / 隱藏 CPA 警示、歷史軌跡、計畫航線等

選項 A 重點：Sidebar 只有「周圍態勢」一個項目；CPA / 歷史 / 計畫航線 / 風險區 / AIS 品質 都是圖層 toggle。 使用者在同一頁面切換看不同東西，視覺 context 永遠不丟（地圖位置、選的船、時間都保留）。 最右側 panel 列船清單，點任一艘可在 panel 內展開歷史 / AIS 品質 / 事件立案。

選項 B：主視角「周圍態勢」+ 子分頁切換

Sidebar 仍只有「周圍態勢」一項，但頁面上方有視角分頁（即時 / 風險 / 歷史 / 品質），切換主視角；底層資料相同

選項 B 重點：Sidebar 也只有「周圍態勢」一個項目；但頁面上方有4 個視角分頁（即時 / 風險 / 歷史 / 品質）。 比 A 更明確區分使用情境——「即時看周圍」「看風險」「回放歷史」「查資料品質」分別有自己的最佳呈現方式， 但底層資料 / 選的船 / 地圖位置都共享 context。

選項比較與建議

兩者都符合「同一個介面 / 同一份資料」原則，差別在使用者切換視角的方式

比較項目選項 A：圖層 toggle選項 B：視角分頁
切換方式 右上多個 toggle 開關，同時看多個圖層 頁面上方分頁，一次看一個視角
同時呈現多種資訊 ✓ 可同時看 CPA + 歷史軌跡 + 計畫航線（疊圖） ✗ 一次只看一個視角，要切換才能比對
視覺複雜度 ⚠ 圖層全開時可能眼花，需考慮優先序 ✓ 每個視角專注一件事，乾淨
學習曲線 ⚠ 新使用者需理解每個 toggle 的意義 ✓ 分頁標籤明確（即時 / 風險 / 歷史 / 品質）
切換成本 ✓ 點一下 toggle 立即看到 / 隱藏 ~ 切分頁需重繪畫面（雖然底層資料不換）
對應使用情境 資深使用者「全圖層疊看」做綜合分析 不同角色 / 任務各看自己視角
對齊既有設計 ✓ 沿用 voyage-alert 的 5 圖層 checkbox 模式 ~ 需新設計分頁元件
實作複雜度 ✓ 較簡單，圖層獨立開關即可 ~ 中等，各視角要設計 panel 內容
建議：採用「選項 A 為主、選項 B 為輔」的混合設計

主介面（選項 A）：地圖 + 圖層 toggle（CPA / 歷史 / 計畫航線 / 風險區 / AIS 品質）—— 對應大部分操作員的日常使用。

子模式（選項 B 元素）：當使用者要「進入歷史回放專屬模式」（需要時間軸控制、倍速）或「進入 AIS 品質專屬模式」（需要熱力矩陣表格）， 再用「全螢幕 mode 切換」進入。

關鍵原則：Sidebar 永遠只有「周圍態勢」一項，不論你切到哪個模式都還是同一個 context；「CPA」「歷史」「AIS 品質」從來不是 sidebar 上的獨立選單。
